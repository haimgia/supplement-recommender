## When to use

Use this skill whenever you need to:

- Create a new health profile.
- Update an existing health profile.
- Delete a health profile.
- Normalize health information from multiple input formats.
- Validate health profile data before it is consumed by other agent skills.

---

## When NOT to use

Do not use this skill to:

- Recommend supplements.
- Retrieve biomedical literature.
- Analyze medication interactions.
- Generate medical advice.
- Interpret laboratory results.

Those responsibilities belong to other skills.

---

## Responsibilities

This skill is responsible for:

- Detecting the input format.
- Parsing structured or unstructured health information.
- Mapping input fields to the canonical health profile schema.
- Validating the normalized profile.
- Creating, updating, or deleting health profiles.
- Returning a normalized profile object and operation status.

---

## Supported Operations

### addProfile

Create a new health profile.

The skill will:

1. Parse the input.
2. Normalize the data.
3. Validate against the canonical schema.
4. Create a new profile.

---

### updateProfile

Modify an existing health profile.

The skill will:

1. Locate the profile using its unique identifier.
2. Normalize incoming fields.
3. Validate the updated profile.
4. Merge valid changes into the existing profile.

Updates should preserve existing information unless explicitly replaced.

---

### deleteProfile

Delete an existing profile.

The skill will:

1. Identify the profile using its unique identifier.
2. Remove the profile from storage.
3. Return a confirmation.

---

## Supported Input Formats

This skill accepts health profile information in any of the following forms:

- Plain text
- JSON
- CSV
- YAML
- XML
- Markdown
- Uploaded files containing health profile information

The skill automatically detects the input format before parsing.

---

## Processing Workflow

For every operation, the skill performs the following steps:

1. Detect the input format.
2. Parse the input into an intermediate representation.
3. Map fields to the canonical schema.
4. Normalize field names and values.
5. Validate against the canonical schema.
6. Execute the requested operation.
7. Return the normalized profile and operation status.

---

## Validation

The canonical schema defined in `references/template.json` is the authoritative health profile model.

Validation includes:

- Required field validation.
- Data type validation.
- Enumeration validation where applicable.
- Field normalization.
- Alias mapping for recognized alternative field names.

Profiles that fail validation are rejected with descriptive validation errors.

---

## Outputs

Successful operations return:

- operation performed
- profile identifier
- normalized health profile
- validation status

Example:

```json
{
  "status": "success",
  "operation": "updateProfile",
  "profileId": "123",
  "profile": { ... }
}
```

Validation failures return:

```json
{
  "status": "error",
  "error": "Validation failed",
  "details": [
    {
      "field": "age",
      "message": "Age must be a positive integer."
    }
  ]
}
```

---

## References

The skill uses the following resources:

- `references/template.json`
  - Canonical health profile schema.
  - Defines the required profile structure.
  - Serves as the authoritative validation model.

If the schema changes, the skill automatically validates against the updated version.

## Notes

This skill maintains a single canonical representation of every health profile.

All downstream skills should consume the normalized profile generated by this skill rather than processing raw user input independently.
